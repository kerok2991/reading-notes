## HTML Chapter 16: Images

This chapter talks about images. Goes over how css can be used to give images a width/height, and that the page will add a placeholder at the given dimensions on the page even if the image hasn't finished loading yet. Useful for getting the page to render with the proper layout even before the image has loaded. Chapter also talks about the hover pseudoclass selector which is used to change the styling based on whether the mouse is hovered over the element. Background images can be added to any div with background-image, and the default is to repeat the image. Chapter also talks about gradients. My personal favorite is the [rainbow background example]("https://developer.mozilla.org/en-US/docs/Web/CSS/gradient/linear-gradient()") in the MDN that stacks three linear gradients. It is perfect.
```css
background: linear-gradient(217deg, rgba(255,0,0,0.8), rgba(255,0,0,0) 70.71%),
            linear-gradient(127deg, rgba(0,255,0,0.8), rgba(0,255,0,0) 70.71%),
            linear-gradient(336deg, rgba(0,0,255,0.8), rgba(0,0,255,0) 70.71%);
```

## HTML Chapter 19: Practical Information

This chapter talks about search engine optimization and webhosting. For search engine optimization, it is important to use appropriate keywords on the website, within the meta tags, and in image alt text. It is also to have links between pages and from external sites with the link labeled in relevant keywords. Having more external sites linking to a page improves priority. The book says it is better to keep keywords organized by their relationship. Only keywords within its field of similar keywords should show on one page. I don't know if this is accurate information but it's a neat concept anyway; that seperation of keywords between pages is better than repeating the all the keywords on all the pages. Brainstorming good keywords is useful, or using helper tools that produce related words. Using analytics it's possible to see which pages users are landing on first and where they are coming from, which could be weak indication of how well a page is search engine optimized.

As far as website hosting goes, it'll need a domain name and a webserver.
