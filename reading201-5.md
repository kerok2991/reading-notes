## HTML Images, CSS Color and Text

### Images

Images are placed onto a page with the `<img>` tag. They speak a billion words very quickly and are useful for setting mood. It is important to set the alt attribute to describe the image in case the image cannot be seen, and the title attribute to give the image a title. Neither are displayed, but a caption can be added by using `<figure>` with the image and `<figcaption>` contained within it. The book discusses a preference for jpeg image format if there are many colors and png and gif for images without many shades or with transparency. It mentions a preference for using pixel dimensions instead of physical dimenions. A displays pixel density may be important to be aware of because of the wide range of pixel densities available now.

Adobe has a color theme picker: [https://color.adobe.com/create/color-wheel](https://color.adobe.com/create/color-wheel), and also has a page for color trends.

### Text

Book talks about text modifiers and types. There's sans serif, serif, monotype, cursive, and fantasy font types. If the font is not available to the browser, it'll go down the list of fonts set by the font-family property until it hits one that is available. If none are available, it should default to the default for that font family, so set "serif" as the last the list to grab whichever is the browsers default font is for that type.

Font sizes are set by the `font-sze` property and can be given units of px, pt, em, or percent. Pt or point is 1/72th of an inch, though I do not know if this is modified by the screen's dpi and user settings. This book was writen before high pixel density screens were popular, so I think the advice to prefer use of px might possibly be outdated, not sure. The following two links may be helpful information [https://learnui.design/blog/ios-font-size-guidelines.html](https://learnui.design/blog/ios-font-size-guidelines.html) and [https://developer.apple.com/design/human-interface-guidelines/ios/visual-design/typography/](https://developer.apple.com/design/human-interface-guidelines/ios/visual-design/typography/). These links discuss dynamic sizes, which takes into account user settings for font size preferences and screen resolution/distance for a given device, but the advice is specifc to apple products and not for css specifically.

Fonts can be important with a file/link using `@font-face {font-family:"name"; src: url('path');}` but keep in mind licensing restrictions. Adobe has a service to embed fonts and takes care of the licensing for the page designer: [https://helpx.adobe.com/fonts/using/add-fonts-website.html](https://helpx.adobe.com/fonts/using/add-fonts-website.html).

In addition to the font properties that modify text (such as text-indent, text-shadow, text-aling, vertical-align), there's also pseudo-elements that use the css selector instead (such as :first-letter, :first-line, :visited, :active, :focus, and :hover). It also lists other selectors to select specific class/identifiers/attributes. For example the substring selector can select the element that has an attribute containing a particular string as its value (eg: `elementTag[attributeName*="stringToSelect"]` would select any tag of that given element that has the specified attribute with a value of stringToSelect).

### Image Formats

This reading is on: [https://blog.imagekit.io/jpeg-vs-png-vs-gif-which-image-format-to-use-and-when-c8913ae3e01d](https://blog.imagekit.io/jpeg-vs-png-vs-gif-which-image-format-to-use-and-when-c8913ae3e01d).

Gives an update on the information contained in the book. It mentions that png and gif are lossless, and that gifs are usually only used for animations now and png is more widely used for lossless images. It mentions jpg has compression and compression artifacts, and therefore are typically smaller in size than gif/png, but for things where compression artificats are a problem, png would probably be a more favorable choice. For most photographs though where some loss is okay, jpeg is preferable. It links a website recommendation that can take an input image and provide the best file format to use (as far as I can tell it doesn't send back the file, instead they host the image on their servers and they handle the optimization on their end): [https://imagekit.io](https://imagekit.io).
