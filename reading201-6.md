### Reading: "[What’s the Difference Between Primitive Values and Object References in JavaScript?](https://betterprogramming.pub/intermediate-javascript-whats-the-difference-between-primitive-values-and-object-references-e863d70677b)"

This talks about how primitive values like strings, numbers, booleans,  null, undefined, bigint, and symbol are immutable (cannot be changed), while objects (such as arrays, functions, and dates) are references and are mutable (can be changed). For example, if a variable is made and a string is placed into it, that variable is the string. On the other hand, if an object is made, there is a reference to it somewhere else in memory. Because these variables are just references, multiple variables can point to the same reference. And if the contents of that reference is changed, it changes for all of the other variables that point to those contents. If the intent is to change only the value of a given variable, not the reference itself, it's probably best to copy the contents of the reference to another location, and point to that new location with a new reference. This also allows changing the contents of both variables without causing changes to the other.

As far as mutability goes, primitives cannot be changed, but they can be completely overwritten (reassigned). For example, the first index of a string cannot be changed because it's immutable, but it can still be completely reassigned to a different string value. On the other hand, references can be changed.

### JS Chapter 3: Object Literals

Objects contain key, value pairs. The keys can either be properties (variables are called properties inside objects) or methods (functions are called methods inside objects). An object can be created using literal notation: `var objectName = {propertyName: value, methodName: function() {functionContents;}}`. The propertName value can be accessed using dot notation: `objectName.propertyName` or using square brackets: `objectName['propertyName']`. Note that later the book discusses that the propertyName can be a variable, a name spaces, or a name that is a number. In the next few pages it talks about constructors, which allow a template for more objects to be created. It uses `this` as a placeholder name for the new object name for use as a placeholder within the constructor.

### JS Chapter 5: Document Object Model

This chapter is on the Document Object Model. So the HTML page is read, a DOM tree with a bunch of nodes/objects is created, and then the page renders. Accessing the DOM objects is like using an API, so there are a bunch of methods that can be used to traverse the position in the tree and add/remove/modify nodes and contents of them. For traversing, it can be done using identifier tags, element names, relative positions to other tags (eg: next-sibling), or css selectors. Since the nodes are objects, they can be added to a variable and later accessed. Also, when getting nodes, it can return a set a nodes. This allows the ability to use loops to go through the entire set. Note that the indices in this set can be accessed either like an array (with brackets) or by using `item()`. To modify content in the tree, there is innerHTML, innerText, and innerValue. There is some security risk to using innerHTML, unless any potential characters are escaped, and links from the user are encoded. There's two general ways to modify content on an HTML page aside from document.write, and that is by using innerHTML/etc., and the other is by modifying the tree with DOM manipulation. For example, it is possible to create a new nodes, add some content to the node, then attach that new node/content to the tree.
