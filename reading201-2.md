## HTML Text, CSS Introduction, and Basic JavaScript Instructions

### HTML Chapter 2: Text Tags
This chapter discusses the difference between structural and semantic text tags. Semantic tags for text indicate some purpose or meaning to the text that they surround. While semantic tags might change the formatting, it is the meaning that is useful. It would be unsuitable to use a `<i>` in place of `<em>` to indicate the content has emphasized meaning, even though by default they both look the same. Semantic tags include: `<strong>`, `<em>`, `<blockquote>`, `<q>`, `<abbr>`, `<cite>`, `<dfn>`,  `<address>`, `<ins>` combined with `<del>`, `<s>`. The book categorizes heading tags as structural tags, as well as `<p>`, `<br>`, `<hr>`, `<sup>`, `<sub>`, `<b>`, `<i>`.

### HTML Chapter 10: CSS Intro
This chapter discusses what CSS is. It adds styling, such as font color, margin sizes, borders, etc., to an HTML page. There's three ways to incorporate CSS. One way is by adding a CSS attribute to any tag, or by linking to the CSS file in the head using a link tag with the href attribute pointing to the file. The last way is putting the CSS directly in the head between style tags.

The precedence order of CSS modifications are important to know. Anything with an "!important" after the selector has priority. Anything with greater specificity has increased importance. And lastly if there are two of the same selectors, the last one takes precedence. 

`.` is the class selector. `#` is the ID selector. `*` is the universal selector that applies to all elements. `a>b` is the child selector and selects only the first child (b). `a b` is the descendant selector, and is the same as the child selector except it applies to all childs and subchildren and so on. `a+b` is the adjacent sibling selector and applies to the closest b after a. `a~b` is the general sibling selector and is the same as the sibling selector except it applies to more than the first sibling if there are more.

### JS Chapter 2: Basic Instructions
This chapter discusses some basics of how to use javascript. It talks about referencing the js file with script tags in the HTML document. It introduces the concept of comments (two backslashes) and statements (each line of code that does something). It mentions variables are like buckets that hold values. These buckets can be created/declared using `var` before the variable name when it is declared. Multiple variables can be declared at the same time, using a single var, and separated by a comma. These values can be numbers, strings, booleans. They can also be arrays containing a set of numbers, strings, and booleans. Arrays can be created as array literals (`[a, b, c]`) or using a constructor (`new Array(a, b, c)`) and that can get stored into a variable. Use `array[index]` access an an element of an array, remembering that the first index is index 0. `Array.length` can be used to find the length of an array; or even a string. Use camelcase for variable names is what the book suggests. A dollarsign is a valid variable name character; dashes are not.

### JS Chapter 4: Flow Control (up to section on Switch Statements)
This chapter talks about flow control using if/else statements and switch statements. If/else statements depend on boolean values to determine whether to enter the if (true) or else (false) cases. Boolean and comparison operations help determine that true or falseness. Two values can be compared using comparison operators, like whether they are equal to each other, or greater than, or greater than or equal, or strictly equal (three equal signs, so matching type is needed in addition to matching value). Two booleans can be compared using boolean operators, such as and (both are true), or (one is true), or not (exlamation point before an expression). Javascript is fairly wishy washy about how explicit types are, and can do some degree of guessing as to what type something is (type coercion). Booleans also have truthy/falsyness to it, so for example an empty string is false, a not empty string is true; zeros is false while any other number is true.
