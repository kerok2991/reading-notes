## HTML Chapter 15: Layout (again)

Repeat reading, so this note only contains the highlights that stood out to me this time around. The highlights to me this time were the pros/cons of fixed width vs fluid pages. I think this is important now with the wide range of dpis/screen sizes, and creating a page that has the versatility for any screen size would be a good thing. For example, reuters.com has a page that has a max width of about 1000px, but when the width of the page shrinks, things that are multiple columns start to collapse into one, and the location of certain pieces moves under the main image rather than to the side. I think airbnb has a neat implementation too; the header image grows almost indefinitely with window width, and as the width of the window decreases, the way it stops shrinking and instead the crops changes from landscape to portait and then the image starts to shrink again; the search bar starts out with multiple filters when it has lots of width, but when the window becomes narrow it collapses into a text search bar. The thumbnail images start out large, but start to shrink, and then the grid starts to collapse into fewer columns, and then finally a horizontal scroll bar appears for that portion of the page (not the entire page, just one portion has a scroll bar). This scroll bar aspect is probably intended more for phones in mind, since it's intuitive to chop off a portion of the thumbnail grid to tease into the remainder of the thumbnail images; and the scroll bar doesn't show in this case, instead it's subconcious almost that the thumbnails scroll horizontally. The book mentions using css templates, like the 960gs grid, which is just a 960 pixel wide grid of 12 columns, and the containers on the page have to be labeled to that templates identifier naming convention. I'm not sure if I'm convinced using a 960gs template is the way to go if the intent is to design fluid width pages though; there might be templates for fluid width pages though, where columns collapse beyond a certain width.

The book mentions position:static is the default usually, and relative positioning can be used to offset away from that default. Absolute is relative to the parent element, and takes that container out of flow, so other containers in the same parent might drop into where that container was if it is suddenly set to absolute. Similar behavior for fixed, except positioning is relative to the window instead of the parent element. Float can be used to automatically push columns to the left in one row, or two, depending on how much vertical space they're given to work within; also useful for getting text to wrap around an image.
