## Local Storage

Fairly outdated article, roughly 10 years old. It talks about local storage before localstorage that came with HTML5 was implemented on every major browser. Before localstorage, either cookies were used to serve a similar purpose, or browser specific methods, or 3rd party tools to handle local storage. The lack of standardization was inconvenient. Localstorage came along and provides users with 5mb of data stored in key/value pairs, where the value is a string. This means that floating point numbers could potentially use up a lot of space since they're stored as strings instead of floats. It also means that anything stored has to be converted from string back to the intended type. If any data in localstorage changes, a localstorage event is triggered which sends with it information on the key that changed, the value that was overwritten, the value that it was changed to, and what url triggered the change. The data in localstorage can be cleared using `localStorage.clear()`; items can be removed using `localStorage.removeItem("key")`; and items can be set or modified using `localStorage.setItem("key", value)`; and items can be read using `localStorage.getItem("key")`. Alternatively bracket notation can be used to get or set (`localStorage["key"]`). The article speculates about the future possibly utilizing sql or some sort of database system, but this article is 10 years old.
